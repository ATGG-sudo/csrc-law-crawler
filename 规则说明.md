# 规则说明

本文记录统一法规目录的分类、匹配、效力判定和替代关系推断规则。规则注册表在 `catalog_rules.py`，包入口为 `csrc_law_crawler.processing.catalog`。代码入口主要在 `amac_crawl.py`、`build_catalog.py`、`normalize_catalog.py` 和 `export_markdown_catalog.py`。

## 规则注册表

每条规则都包含：

- `rule_id`：稳定、可审计的规则编号。
- `description`：规则说明。
- `confidence`：默认置信度。
- `confidence_band`：由统一阈值映射出的置信度分层。
- `evidence_fields`：规则判断依赖的证据字段。

规则清单会写入：

- `work/catalog/manifest.json` 的 `rules`。
- `reports/review_queue.json` 的 `rules`。

同时会写入 `rule_calibration`，其中包含：

- `confidence_bands`：统一置信度分层，当前为 `certain >= 0.99`、`strong >= 0.90`、`moderate >= 0.75`、`low < 0.75`。
- `source_match_review_confidence_threshold`：来源匹配进入人工复核的置信度阈值，当前为 `0.95`。

AMAC 源记录会在 metadata 中写入 `document_type_rule_id`。统一目录匹配会在 `canonical/indexes/source_map.json` 中写入 `match_rule_id`。关系边会在 `work/relations/catalog_relations.json` 中写入顶层 `rule_id` 和 evidence 内的 `rule_id`。

## AMAC 分类

AMAC 页面和附件使用集中规则分类：

- `classification.amac_publication_notice`：标题为发布公告或通知。
- `classification.amac_regulatory_practice`：登记备案动态或 `/dbdt/` 页面。
- `classification.amac_self_regulatory_rule`：标题包含办法、规则、指引、细则等制度词。
- `classification.amac_supporting_material`：没有更强规则命中时归为辅助材料。

`classify_document(title, url)` 保持旧接口返回 `document_type`，内部由 `catalog_rules.classify_amac_document()` 返回分类和规则对象。

## 来源匹配

AMAC 与 NERIS 合并使用以下匹配规则：

- `match.neris_title_absent`：NERIS 中没有同规范化标题记录，视为 AMAC 新增来源。
- `match.normalized_title_fileno`：规范化标题和文号一致，视为同一文件。
- `match.normalized_title_pub_date_window`：规范化标题一致且发布日期相差不超过 3 日。
- `match.unique_normalized_title`：只有一个同题名 NERIS 候选，但日期或文号证据不足。
- `match.amac_internal_title_date`：多个 AMAC 页面或附件题名、日期一致，归并为同一实体。
- `match.ambiguous_normalized_title`：存在多个同题名 NERIS 候选，进入人工核验队列。

低于 `source_match_review_confidence_threshold` 的非歧义来源匹配也会进入人工核验队列，原因标记为 `source_match_low_confidence`，并记录 `match_rule_id` 与 `match_confidence`。

## 效力状态

统一目录保留来源原始 `status`，并额外生成 `effectiveness`：

- `current`：现行有效，或 AMAC 正式制度未显式标注效力但没有发现排除证据。
- `historical`：已废止、失效、被修改，或被后续正式版替代。
- `not_applicable`：征求意见稿、公告、动态、说明、模板等仅供参考材料。
- `unknown`：证据不足，需要人工核验。

`effectiveness` 同时写入：

- `raw_status`：来源原始效力状态。
- `label`：中文可读标签。
- `basis`：判定依据，例如 `explicit_current_status`、`comment_draft_signal`、`amac_official_rule_default`、`superseded_by_catalog_relation`。
- `rule_id`：可审计规则编号，例如 `effectiveness.explicit_current_status`、`effectiveness.amac_official_rule_default`。
- `confidence`：规则置信度。
- `superseded_by`：如被替代，记录替代文件及证据。

## 判定优先级

1. 来源明确标注已废止、失效、已被修改等，归为 `historical`。
2. 标题、名称或正文出现征求意见稿信号，归为 `not_applicable`，标签为“征求意见/仅供参考”。
3. 来源明确标注“现行有效”，归为 `current`。
4. catalog 推断出后续正式版替代试行版，旧试行版归为 `historical`。
5. 公告、动态、说明、模板等参考材料归为 `not_applicable`。
6. AMAC 正式制度缺少显式效力状态时，默认归为 `current`，依据为 `amac_official_rule_default`，规则编号为 `effectiveness.amac_official_rule_default`。
7. 其他情况归为 `unknown`。

## 征求意见稿

以下信号会把文件识别为仅供参考：

- `征求意见稿`
- `公开征求意见`
- `征求意见的通知`
- `征求意见通知`
- `草案`

该类文件不进入 `canonical/markdown/current/`，而是进入 `canonical/markdown/reference/`。

## 参考材料

即使标题中含有“办法”“规则”等制度词，以下材料也不按现行制度处理：

- 参考模板
- 修订说明
- 起草说明
- 填写说明
- 说明材料
- 问题解答、业务问答
- 解读、培训材料

该类文件归为 `not_applicable`，依据为 `reference_title_signal` 或 `reference_document_type`。

## AMAC 缺省效力

AMAC 官网不少正式制度没有显式效力状态，因此缺少 `现行有效` 不等于无效。

当文件满足以下条件时，归为 `current`：

- 首选来源为 AMAC。
- 文件类型为 `regulation` 或 `self_regulatory_rule`。
- 原始状态为空或 `unknown`。
- 没有征求意见稿信号，也没有被替代证据。
- 没有参考模板、修订说明、问答、解读、培训等参考材料信号。

该规则不会应用于公告、动态、说明、模板、培训通知等参考材料。

## 试行制度

`试行` 本身不代表无效。试行制度默认按正式制度处理。

当同时满足以下条件时，推断后续正式版替代试行版：

- 旧文件标题含 `试行`。
- 新文件标题不含 `试行`。
- 去除 `试行` 和标点后的规范化标题一致。
- 新文件发布日期晚于旧文件。
- 两者都是 `regulation` 或 `self_regulatory_rule`。
- 若两者都有发布机构，发布机构必须一致。

推断成功后生成：

- `canonical/relations/graph.json` 中的 `supersedes` 边。
- `work/relations/catalog_relations.json` 中来源为 `catalog.trial_replacement` 的证据。
- `work/relations/catalog_relations.json` 中的 `rule_id = relation.trial_replacement.same_title_later_formal`。
- 旧试行版 normalized JSON 中的 `effectiveness.status = historical`。

## 人工核验

自动规则是检索和归档前置处理，不替代法律判断。正式使用前仍应回到官方页面核验文件完整性、发布时间、效力状态和是否存在后续修订。

`reports/review_queue.json` 目前会收集：

- `source_match_ambiguous`：AMAC 与 NERIS 来源匹配歧义。
- `source_match_low_confidence`：来源匹配不歧义，但置信度低于统一人工复核阈值。
- `effectiveness_unknown`：AMAC 正式制度缺少明确效力状态，需要人工复核。
